## 1.5. 摘要
随着电路设计复杂度的不断增长，多 FPGA 系统在逻辑仿真和快速原型验证中变得越来越流行。在多 FPGA 系统中，不同的 FPGA 通过有限的物理线路连接，也就是说，一个 FPGA 通常只与少数几个 FPGA 有直接连接。在电路划分阶段，如果将两个直接相连的节点分配到两个没有物理链路的 FPGA 上，将显著增加延迟并降低整体性能。然而，一些著名的划分工具，如 hMETIS 和 PaToH，主要关注最小化割规模，而没有考虑这种 FPGA 的拓扑约束，这限制了它们的实际应用。在本文中，我们提出了一个多级拓扑驱动划分框架，名为 TopoPart，用于处理多 FPGA 系统中的拓扑约束。具体而言，我们首先在粗化阶段设计了一种候选 FPGA 传播算法，以保证后续阶段不出现拓扑违规。在最后的细化阶段，在保持拓扑和资源约束的同时，迭代优化割规模。与提出的基线相比，我们的划分算法实现了零拓扑违规，同时给出了更小的割规模。

# 2. 整体概括

## 2.1. 研究背景与动机
### 2.1.1. 核心问题
随着现代集成电路设计规模的指数级增长，单个 FPGA 往往无法容纳整个设计，因此需要使用 <strong> 多 FPGA 系统 (Multi-FPGA Systems, MFS)</strong>。在 MFS 中，电路被划分成多个子电路，分别映射到不同的 FPGA 上。然而，FPGA 之间的物理互连资源是有限的，且通常呈现稀疏的、不规则的拓扑结构。传统的电路划分算法（如 hMETIS）主要致力于最小化 <strong> 割规模 (Cut Size)</strong>，即被切断的连线数量，却忽略了 FPGA 之间的物理连接限制。如果两个在逻辑上紧密连接的电路节点被分配到了两个物理上不直接相连的 FPGA 上，信号就需要通过中间的 FPGA 进行 <strong> 路由跳数 (Routing Hops)</strong> 转发。这不仅增加了信号延迟，还占用了宝贵的互连带宽，导致系统性能严重下降。

### 2.1.2. 现有挑战与空白
现有的主流划分工具（hMETIS, PaToH）虽然能很好地处理平衡约束和固定节点约束，但它们无法感知底层的硬件拓扑。一些现有的工作虽然考虑了拓扑，但主要集中在路由阶段而非划分阶段，或者无法同时处理固定节点约束。因此，当前领域缺乏一种能够同时满足以下三个约束的高效划分算法：
1.  **拓扑约束**：直接相连的电路节点必须位于同一 FPGA 或物理直连的 FPGA 上。
2.  **资源约束**：每个 FPGA 上的资源占用不能超过其容量。
3.  **固定节点约束**：某些特定模块必须预先分配到指定的 FPGA。

### 2.1.3. 切入点
本文作者提出了一种全新的 <strong> 多级拓扑驱动划分框架 (Multi-level Topology-Driven Partitioning Framework)</strong>。其核心切入点在于：在划分的早期阶段（粗化阶段之前），就通过算法计算出每个电路节点允许被放置的“候选 FPGA 集合”，并将这一信息贯穿于整个多级划分流程（粗化、初始划分、解粗化、细化）中，从而在保证零拓扑违规的前提下优化割规模。

## 2.2. 核心贡献/主要发现
1.  **首个多级拓扑感知框架**：提出了第一个同时尊重多 FPGA 系统拓扑约束和固定节点需求的多级划分框架。
2.  **候选 FPGA 传播算法**：设计了一种新颖且高效的算法，基于图论距离定理，初始化并维护候选 FPGA 信息，从理论上保证了后续划分过程不会产生拓扑违规。
3.  **实验验证**：在生成的基准测试集上的实验表明，TopoPart 能够在满足所有约束（包括固定节点）的同时，实现 <strong> 零拓扑违规 (Zero Topology Violation)</strong>，并且相比于 PaToH 结合模拟退火的基线方法，还能进一步减少割规模。

# 3. 预备知识与相关工作

## 3.1. 基础概念
为了理解本文，我们需要掌握以下 EDA 领域的基础概念：

- **超图与超图划分**：在 VLSI 设计中，电路通常被建模为超图，其中顶点代表逻辑单元（如 LUT、FF），超边代表线网，连接多个引脚。划分的目标是将顶点分配到不同的块中，使得被切断的超边（即跨越不同块的线网）数量最少。
- **多级划分框架**：这是一种处理大规模图划分的标准范式，包含三个阶段：
    1.  **粗化**：通过合并连通的顶点，将大图逐渐收缩成一个小图。
    2.  **初始划分**：在最小的图上进行划分，通常使用贪心或随机算法。
    3.  **解粗化与细化**：将图逐级展开回原始大小，并在每一级通过移动顶点来优化划分质量。
- **割规模**：衡量划分质量的主要指标，指跨越不同划分块的连线总数。割规模越小，意味着块间通信越少，通常布线压力越小。
- **拓扑约束**：在多 FPGA 系统中，指 FPGA 之间的物理连接关系。如果两个逻辑节点在电路图中相连，它们被分配到的 FPGA 在物理图中也必须是相连的（或为同一 FPGA），否则需要通过多跳路由，这被称为拓扑违规。
- **固定节点**：在某些设计中，由于 I/O 接口位置限制或特定功能需求，某些逻辑单元必须被放置在特定的 FPGA 上，这些节点在划分过程中不可移动。

## 3.2. 前人工作
作者在引言中提到了以下关键前人工作：
- **hMETIS**：经典的基于多级框架的图/超图划分工具，主要关注最小化割规模和保持平衡，但不支持拓扑约束。
- **PaToH**：另一种流行的超图划分工具，支持固定节点约束，但同样不支持拓扑约束。
- **Farooq et al. [6]**：考虑了多 FPGA 系统的拓扑，但其主要工作集中在 FPGA 间的路由阶段，而非电路划分阶段。
- **Liou et al. [1]**：研究通过更新线网权重并结合 hMETIS 来解决多 FPGA 划分问题，主要关注时序和 TDM（时分复用）。
- **Chen et al. [12]**：提出同时进行划分和信号分组以减少信号延迟，但未考虑物理拓扑。

## 3.1. 技术演进
多 FPGA 划分技术的发展经历了从单纯的 **最小化割规模**，到考虑 **资源异构性**，再到考虑 **时序驱动** 的过程。然而，随着系统互连带宽成为瓶颈，学术界和工业界开始关注 **拓扑感知**。本文的工作处于这一技术脉络的前沿，首次将严格的拓扑约束深度集成到多级划分算法的内部逻辑中，而非作为后处理步骤。

## 3.2. 差异化分析
本文方法与现有工作（特别是作为基线的 PaToH + 模拟退火）的核心区别在于：
- **集成 vs. 后处理**：基线方法首先使用 PaToH 忽略拓扑进行划分，然后使用模拟退火试图修复拓扑违规。这种“先污染后治理”的方式往往难以消除所有违规，且修复过程会破坏原有的割规模优化。而 TopoPart 将拓扑约束作为 **硬约束**，通过候选 FPGA 集合在算法内部进行严格限制，从源头杜绝违规。
- **理论保证**：本文提出的候选 FPGA 传播算法基于定理 III.1，提供了可行性的理论保证，而启发式的后处理方法无法提供这种保证。

# 4. 方法论

## 4.1. 方法原理
TopoPart 的核心思想是利用 **多级框架** 处理大规模电路，同时引入 **候选 FPGA** 的概念来处理拓扑约束。其背后的直觉是：如果一个电路节点距离某个固定节点很近（在电路图上），那么它只能被放置在距离该固定节点对应的 FPGA 也很近（在 FPGA 物理图上）的位置上，否则信号路径将无法在有限的跳数内完成路由。

算法流程主要包含四个步骤：
1.  **候选 FPGA 传播**：利用固定节点和拓扑信息，计算每个可移动节点允许放置的 FPGA 集合。
2.  **候选 FPGA 驱动的粗化**：在合并节点时，不仅考虑连通性，还要求合并的节点共享足够多的候选 FPGA。
3.  **拓扑驱动的初始划分**：在粗化后的最小图上，利用贪心策略和回溯机制生成满足所有约束的初始解。
4.  **解粗化与细化**：逐级展开图，并通过移动边界节点来优化割规模，同时始终保持不违反约束。

## 4.2. 核心方法详解 (逐层深入)

### 4.2.1. 问题定义
首先，作者形式化了拓扑驱动划分问题。给定电路图 `G(E, V)` 和 FPGA 物理图 $\hat{G}(\hat{E}, \hat{V})$，目标是找到划分 $P$，最小化割规模，同时满足：
1.  **固定节点约束**：固定节点 $v_i \in V_\gamma$ 必须分配到指定 FPGA。
2.  **资源约束**：每个 FPGA $\hat{v}_i$ 上的节点数 $|p_i| \le r_i$。
3.  **拓扑约束**：对于任意电路边 $(v_i, v_j) \in E$，要么 $part(v_i) = part(v_j)$（在同一 FPGA），要么 $(part(v_i), part(v_j)) \in \hat{E}$（在物理直连的 FPGA 上）。

### 4.2.2. 候选 FPGA 传播
这是 TopoPart 的核心创新点。为了满足拓扑约束，算法首先计算每个节点 $v_i$ 可以被分配的 FPGA 集合 $Cddt(v_i)$。

**理论基础：**
作者提出了一个关键定理来指导候选 FPGA 的计算。

**定理 III.1**
设 $v_i$ 和 $v_j$ 在电路图 $G$ 上的最短路径长度为 $x$，$\hat{v}_i$ 和 $\hat{v}_j$ 在 FPGA 图 $\hat{G}$ 上的最短路径长度为 $y$。$v_i$ 和 $v_j$ 可以分别无拓扑违规地分配给 $\hat{v}_i$ 和 $\hat{v}_j$ 的充要条件是：
$x \geq y$
*符号解释：*

- $v_i, v_j$: 电路图中的节点。
- $\hat{v}_i, \hat{v}_j$: FPGA 图中的节点。
- $x$: 电路节点间的逻辑距离。
- $y$: FPGA 节点间的物理距离。

  这个定理的直观含义是：电路上的逻辑距离必须足够长，以“容纳”物理上的距离。如果物理距离 $y$ 很大（FPGA 离得很远），那么电路上的逻辑距离 $x$ 必须更大，否则信号无法在物理上路由过去。

**算法实现：**

在现有 FM 算法的基础上进行改进，首先进行传播过程计算，需要得到计算：FPGA 图中所有节点对的最短路径、每个节点最大距离 $maxDist(\hat{v}_i)$。在传播过程中从现有固定节点对出发使用 BFS 在电路图中搜索符合要求的可移动节点，再利用 dist 取交集。如果更新后 $Cddt(v_j)$ 只剩下一个 FPGA，则 $v_j$ 变为新的固定节点，加入队列继续传播。直到传播结束此时进入初始化划分

初始化划分：这是一个贪心算法，带有回溯机制。

1.  **优先级队列**：维护两个优先队列。队列 $Q$ 决定节点的处理顺序（候选 FPGA 越少越优先，因为它们受限最多）；队列 $R_j$ 决定节点 $v_j$ 分配到哪个 FPGA（优先选择能带来最小割规模增量的 FPGA）。
2.  **贪心分配与回溯**：
    - 每次从 $Q$ 中取出候选数最少的节点 $v_j'$，尝试将其分配到 $R_j$ 中最好的 FPGA $\hat{v}_j$。
    - 分配后，检查所有未分配的邻居节点 $v_k'$。根据拓扑约束，$v_k'$ 的候选 FPGA 中不能包含与 $\hat{v}_j$ 不直连的 FPGA。因此，更新 $Cddt(v_k')$。
    - **关键点**：如果某个邻居 $v_k'$ 的候选集变为空，说明当前分配导致死锁。此时触发**回溯**：撤销 $v_j'$ 的分配，并将 $\hat{v}_j$ 从 $v_j'$ 的候选集中永久移除，然后尝试下一个候选 FPGA。
    - 为了高效支持回溯，算法使用了一个计数向量 $T_i$ 来记录候选 FPGA 的引用次数，从而在 $O(m)$ 时间（$m$ 为 FPGA 数量）内完成更新和恢复。



基于上述定理，作者设计了 `Algorithm 1: Candidate FPGA Propagation`。

1.  **预处理**：使用 Dijkstra 算法计算 FPGA 图中所有节点对的最短路径，并计算每个 FPGA 节点 $\hat{v}_i$ 到其他节点的最大距离 $maxDist(\hat{v}_i)$。
2.  **初始化**：对于固定节点 $v_i$，其候选集合 $Cddt(v_i)$ 仅包含其指定的 FPGA。对于可移动节点，初始化为全集 $\hat{V}$。
3.  **传播过程**：使用一个队列 $Q$ 存储已确定的节点对 $(v_i, \hat{v}_i)$。对于队列中的每个节点对，算法检查其影响范围内的所有可移动节点 $v_j$。
    
    - 如果电路距离 $dist(v_j, v_i) < maxDist(\hat{v}_i)$，说明固定节点 $v_i$ 对 $v_j$ 的位置有限制作用。
    - 此时，$v_j$ 的候选 FPGA 集合 $Cddt(v_j)$ 需要更新为原集合与 $\hat{S}(\hat{v}_i, k)$ 的交集，其中 `k = dist(v_j, v_i)`，$\hat{S}(\hat{v}_i, k)$ 是 FPGA 图中距离 $\hat{v}_i$ 不超过 $k$ 的节点集合。
    - 如果更新后 $Cddt(v_j)$ 只剩下一个 FPGA，则 $v_j$ 变为新的固定节点，加入队列继续传播。
    - 如果 $Cddt(v_j)$ 为空，则说明无解。

### 4.2.3. 候选 FPGA 驱动的粗化
传统的粗化策略只看重节点间的连通性（权重）。但在 TopoPart 中，如果两个节点虽然连通性强，但它们的候选 FPGA 集合交集很小，强行合并会导致合并后的超级节点候选集合过小，甚至为空，从而导致后续划分无解。

因此，作者引入参数 $\beta$ 来控制粗化。只有当两个节点满足以下条件时才合并：
1.  连通性高。
2.  公共候选 FPGA 的数量 $|Cddt(v_i) \cap Cddt(v_j)| \ge \beta$。

    文中 $\beta$ 初始设为 $\frac{2}{3} |\hat{V}|$。如果在初始划分阶段发现无解，算法会回退并增大 $\beta$ 重新粗化，这是一个保守的策略。

### 4.2.5. 解粗化与细化
得到初始解后，算法进入解粗化和细化阶段。
1.  **展开**：将超级节点逐步展开为原始节点，继承上一级的划分结果。
2.  **细化**：在每一级，识别边界节点（即与其他 FPGA 有连接的节点）。尝试将这些节点移动到其他候选 FPGA 中。
3.  **移动策略**：只有当移动节点 $v_i$ 到新 FPGA 能减少割规模，且不违反资源约束和拓扑约束时，才执行移动。由于候选 FPGA 集合的存在，拓扑约束的检查变得非常高效。

## 5.2. 评估指标
论文使用了以下两个主要评估指标：

1.  <strong> Cut Size (割规模)</strong>
    -   **概念定义**：指划分方案中被切断的线网数量。这是衡量划分质量的核心指标，割规模越小，表示跨 FPGA 的通信越少，通常意味着更好的时序性能和更低的布线拥塞。
    -   **数学公式**：对于超图 $H=(V, E)$ 和划分块集合 $\{V_1, ..., V_k\}$，割规模定义为：
        $$ \text{CutSize} = \sum_{e \in E} \left[ \exists i, j, i \neq j, \text{such that } e \cap V_i \neq \emptyset \text{ and } e \cap V_j \neq \emptyset \right] $$
        *符号解释：*
        - $E$: 超边（线网）集合。
        - $e$: 单个超边。
        - $V_i, V_j$: 不同的划分块（FPGA）。
        - $[\cdot]$: Iverson bracket，条件为真时取 1，否则取 0。

2.  <strong># Violations (拓扑违规数)</strong>
    -   **概念定义**：指在划分结果中，不满足拓扑约束的线网数量。即两个逻辑上相连的节点被分配到了物理上不直接相连的 FPGA 上的情况。
    -   **数学公式**：
        $$ \text{Violations} = \sum_{(u,v) \in E_{2pin}} \left[ part(u) \neq part(v) \land (part(u), part(v)) \notin \hat{E} \right] $$
        *符号解释：*
        - $E_{2pin}$: 转换后的二端线网集合。
        - `part(u)`: 节点 $u$ 被分配到的 FPGA。
        - $\hat{E}$: FPGA 物理图中的边集合（表示直连关系）。

## 5.3. 对比基线
由于没有现成的支持拓扑约束的工具，作者设计了一个合理的基线：
-   <strong> PaToH + Simulated Annealing (模拟退火)</strong>：
    1.  使用 PaToH 进行初始划分（仅考虑固定节点，忽略拓扑）。
    2.  使用模拟退火算法进行后处理，迭代地移动节点以减少拓扑违规。
-   <strong> Reference Solution (参考解)</strong>：
    -   作者通过简单的迭代移动生成的可行解，保证零违规，但割规模通常很大。这主要作为一个上限参考。

# 6. 实验结果与分析

## 6.1. 核心结果分析
实验在两个不同的多 FPGA 系统（MFS1 和 MFS2）上进行，主要结论如下：

1.  **零拓扑违规**：TopoPart 在所有测试用例中均实现了 **# Violations = 0**，完美满足拓扑约束。相比之下，基线方法在 MFS1 上仍有约 13% 的割边违规，在更稀疏的 MFS2 上更是高达 71% 的割边违规。这证明了后处理方法难以彻底解决拓扑冲突。
2.  **更小的割规模**：尽管 TopoPart 需要严格满足拓扑约束，但其割规模依然优于基线。
    -   在 MFS1 上，TopoPart 的平均割规模仅为基线的 84%（即减少了 16%）。
    -   在 MFS2 上，TopoPart 的平均割规模为基线的 79%。
        这表明将拓扑约束集成到划分算法内部，比先忽略再修复能获得更优的全局解。
3.  **运行时间**：TopoPart 的运行时间比基线长（MFS1 上平均约 4.85 倍，MFS2 上约 6.32 倍）。作者指出这是为了保证拓扑合规性所付出的代价，且时间仍在可接受范围内（最长约 1200 秒）。

## 6.3. 消融实验/参数分析
论文中主要涉及的一个关键参数是粗化阶段的公共候选 FPGA 阈值 $\beta$。
- **参数 $\beta$ 的影响**：作者指出 $\beta$ 是效率与质量的权衡。如果 $\beta$ 太大，粗化效果差，导致解空间搜索效率低；如果 $\beta$ 太小，容易导致合并后的节点候选集过小，使得初始划分无解。
- **自适应策略**：TopoPart 采用了一种自适应策略。如果初始划分失败（无解），算法会自动增大 $\beta$ 并重新进行粗化和划分。这保证了算法的鲁棒性，虽然可能增加运行时间，但能找到可行解。

# 7. 总结与思考

## 7.1. 结论总结
本文针对多 FPGA 系统中物理拓扑连接受限的问题，提出了 TopoPart 框架。这是首个将拓扑约束深度集成到多级划分流程中的工作。通过引入候选 FPGA 传播算法和改进的粗化/初始划分策略，TopoPart 能够在保证零拓扑违规和满足固定节点约束的同时，获得比传统后处理方法更小的割规模。实验结果有力地证明了在硬件约束敏感的划分任务中，将约束内嵌到算法核心逻辑中优于外挂式的修复方法。

## 7.2. 局限性与未来工作
- **运行时间**：作者承认 TopoPart 的运行时间比基线方法（PaToH + SA）要长。虽然对于离线划分任务通常可以接受，但在需要快速迭代的场景下可能存在挑战。
- **复杂拓扑**：目前的算法基于 FPGA 图和电路图的最短路径。对于包含复杂交换网络或动态路由路径的多 FPGA 系统，如何定义和计算“距离”以及“拓扑约束”可能需要进一步扩展。
- **资源类型单一**：文中主要考虑了节点数量作为资源约束。实际 FPGA 中包含 LUT、FF、DSP 等多种资源，如何处理多维资源约束下的拓扑划分是一个潜在方向。

## 7.3. 个人启发与批判
- **方法论启发**：本文提出的“候选集合传播”思想非常巧妙。它将复杂的全局拓扑约束转化为局部的节点属性约束，从而使得后续的贪心或局部搜索算法能够简单有效地执行。这种“约束预处理”的策略在解决组合优化问题时具有很高的通用价值。
- **批判性思考**：
    - **基准测试的局限性**：由于缺乏公开的带有拓扑约束的标准基准，作者使用了合成数据。虽然合成数据构建逻辑合理，但可能与真实工业界电路的拓扑特性存在差异。未来工作若能包含真实的多 FPGA 设计案例将更具说服力。
    - **固定节点的影响**：算法严重依赖固定节点来启动传播过程。如果固定节点很少或者分布极不均匀，传播算法的效果可能会减弱，候选集合可能收敛得很慢或无法收敛到足够小的集合，从而影响粗化效率。
    - **扩展性**：该方法的核心在于距离计算和集合操作。对于超大规模的系统（如数百个 FPGA），FPGA 图上的全源最短路径计算（APSP）可能会成为预处理的瓶颈，虽然 $N=43$ 时尚可，但 $N=1000$ 时可能需要更高效的距离近似算法。
